# Bysheen Cross-Border Product Agent System

## 多 Agent 协同的跨境商品内容生产系统

Bysheen Cross-Border Product Agent System 是一个面向跨境电商卖家的 AI 商品内容生产系统，主要用于将 1688 等中文货源平台的商品信息，自动转化为适合 TikTok Shop、Shopee、Lazada 等海外电商平台上架的本地化商品资料。

本项目通过多 Agent 协同工作流，实现商品采集、内容清洗、多语言翻译、图片检测、图片优化、平台合规检查和 TikTok Shop 批量上架 Excel 导出，帮助跨境卖家降低人工搬运商品的时间成本，提高商品上架效率，减少翻译、修图、填表和合规检查中的错误。

---

# 一、项目解决的核心痛点

跨境电商卖家在从 1688、淘宝、拼多多等国内货源平台搬运商品到 TikTok Shop 等海外平台时，通常需要完成商品采集、标题翻译、详情翻译、图片处理、SKU 整理、价格整理、平台模板填写等多个步骤。

传统流程主要存在以下痛点：

## 1. 商品采集效率低

运营人员需要手动打开商品页面，复制商品标题、价格、规格、主图、详情图和商品描述，再整理成平台要求的格式。

一个商品从采集到整理完成通常需要 20 到 30 分钟。如果需要批量铺货几十个、几百个商品，人工处理效率非常低。

## 2. 中文商品内容不适合海外平台

1688 商品标题和详情页大多是中文，并且包含大量国内电商表达方式，例如厂家直销、批发、包邮、源头工厂、现货供应等。

这些内容如果直接翻译成英文，往往会出现语言生硬、关键词不准确、卖点不清晰、不符合当地消费者搜索习惯等问题。

## 3. 商品图片不符合跨境平台要求

很多 1688 商品图片包含中文文字、水印、厂家信息、联系方式、二维码、营业执照、资质证书、工厂照片等内容。

这些图片如果直接上传到 TikTok Shop，可能会影响平台审核，也会降低海外消费者的信任感和购买转化率。

## 4. TikTok Shop 上架模板字段复杂

TikTok Shop 批量上传模板通常包含标题、描述、类目、品牌、SKU、规格、价格、库存、重量、尺寸、图片链接等多个字段。

人工填写 Excel 模板容易出现字段缺失、格式错误、图片链接错误、SKU 对应错误等问题。

## 5. 多国家、多店铺运营成本高

跨境卖家通常需要同时运营多个国家站点，例如新加坡、马来西亚、菲律宾、泰国、越南、美国等。

不同国家站点需要不同语言、不同标题风格、不同价格策略和不同平台规范。如果完全依赖人工处理，运营成本非常高。

# 三、系统核心逻辑

本项目不是一个简单的 AI 问答工具，而是一个完整的长链路 AI Agent 工作流。

系统整体流程如下：

商品输入  
↓  
采集 Agent  
↓  
清洗 Agent  
↓  
翻译 Agent  
↓  
图片 Agent  
↓  
合规 Agent  
↓  
导出 Agent  
↓  
TikTok Shop 批量上架 Excel  

整个流程的目标是把一个原始中文商品，自动加工成一个可以直接用于跨境平台上架的标准化商品资料。

# 四、六大核心 Agent 设计

## 1. 采集 Agent

采集 Agent 负责从商品页面、商品链接或用户输入中提取基础商品信息。

主要任务包括：

- 提取商品标题
- 提取商品价格
- 提取商品规格
- 提取商品 SKU
- 提取商品主图
- 提取商品详情图
- 提取商品描述
- 提取商品属性
- 将非结构化商品数据转化为结构化 JSON 数据

采集 Agent 的目标是把原始商品信息整理成系统可以继续处理的标准数据。

---

## 2. 清洗 Agent

清洗 Agent 负责对采集到的商品内容进行过滤、清理和标准化。

主要任务包括：

- 删除无效字段
- 删除重复图片
- 过滤低质量图片
- 识别工厂资质图
- 识别二维码图片
- 识别联系方式图片
- 识别非商品相关图片
- 清理无效描述
- 整理商品规格
- 提取真正适合上架的商品卖点

清洗 Agent 的作用是把混乱的原始商品资料变成干净、可处理、适合 AI 翻译和平台上架的数据。

---

## 3. 翻译 Agent

翻译 Agent 负责将中文商品内容转化为目标市场语言。

主要任务包括：

- 中文标题翻译为英文
- 商品卖点本地化改写
- 商品描述多语言生成
- 根据目标国家优化表达方式
- 根据跨境电商搜索习惯生成关键词
- 避免直译、生硬翻译和无效关键词堆砌
- 生成更适合 TikTok Shop 搜索和转化的商品标题

示例：

原始标题：

厨房不锈钢多层置物架家用落地收纳架

优化后标题：

Stainless Steel Multi-Layer Kitchen Storage Rack for Home Organization

翻译 Agent 不只是翻译文字，而是要根据目标市场进行本地化改写，让商品内容更符合海外消费者的搜索和购买习惯。

---

## 4. 图片 Agent

图片 Agent 负责处理商品图片相关任务。

主要任务包括：

- OCR 识别图片中文字
- 检测图片水印
- 检测中文营销字
- 检测二维码
- 检测厂家联系方式
- 检测低清晰度图片
- 检测非商品相关图片
- 给商品主图打分
- 推荐最佳商品主图
- 标记需要重做的图片
- 为后续生成场景图、营销图提供基础数据

图片 Agent 的核心价值是避免卖家直接上传不适合海外平台的 1688 原图，提高商品图片的合规性和转化率。

---

## 5. 合规 Agent

合规 Agent 负责检查商品是否符合 TikTok Shop 等跨境电商平台的基础上架要求。

主要任务包括：

- 检查标题长度
- 检查敏感词
- 检查违禁词
- 检查类目是否匹配
- 检查图片是否可能影响审核
- 检查商品描述是否完整
- 检查 SKU 是否完整
- 检查价格字段是否完整
- 检查库存字段是否完整
- 输出商品合规评分
- 输出需要人工复核的问题

合规 Agent 的目标是在商品正式导出或上架之前，提前发现可能影响审核和转化的问题。

---

## 6. 导出 Agent

导出 Agent 负责将最终处理好的商品数据生成平台可用文件。

主要任务包括：

- 生成 TikTok Shop 批量上传 Excel
- 按平台字段映射商品数据
- 自动填充标题
- 自动填充描述
- 自动填充价格
- 自动填充库存
- 自动填充 SKU
- 自动填充规格
- 自动填充图片链接
- 检查必填字段
- 输出可下载文件
- 为后续自动上架流程提供标准数据

最终输出文件可以作为 TikTok Shop 批量上架模板使用。

# 五、是否包含长链推理

本项目包含长链推理能力。

原因是系统不是简单地调用一次 AI 生成标题，而是需要根据商品信息、目标市场、图片质量、平台规则、SKU 信息和合规要求进行连续判断。

例如，一个商品进入系统后，AI 需要依次判断：

1. 这个商品是否适合跨境销售
2. 商品标题应该保留哪些核心关键词
3. 哪些中文词需要翻译
4. 哪些词需要本地化改写
5. 哪些图片可以作为主图
6. 哪些图片包含中文、水印或厂家信息
7. 商品描述是否符合海外消费者阅读习惯
8. 标题长度是否符合 TikTok Shop 规则
9. 商品类目是否匹配
10. 商品信息是否可以导出到官方模板
11. 是否需要人工复核

因此，本系统属于长链路、多步骤、多判断的 AI Agent 工作流。

长链推理体现在：

- 多步骤任务拆解
- 多轮数据判断
- 多 Agent 之间的数据传递
- 根据前一步结果决定下一步动作
- 根据商品类型选择不同处理策略
- 根据图片质量决定是否进入优化流程
- 根据合规结果决定是否允许导出
- 根据目标国家决定语言和标题风格

# 六、是否包含多 Agent 协作

本项目包含多 Agent 协作。

不同 Agent 不直接混在一起处理所有任务，而是按照职责分工协作。

Agent 分工如下：

| Agent | 主要职责 |
|---|---|
| 采集 Agent | 获取商品原始数据 |
| 清洗 Agent | 过滤无效内容 |
| 翻译 Agent | 多语言翻译和本地化 |
| 图片 Agent | OCR、图片检测、主图评分 |
| 合规 Agent | 平台规则检查 |
| 导出 Agent | 生成上架 Excel 文件 |

每个 Agent 接收上一个 Agent 的输出，并生成下一个 Agent 所需的数据。

这种设计的优势包括：

- 逻辑清晰
- 方便扩展
- 方便替换不同 AI 模型
- 方便定位错误
- 方便统计每个环节的 Token 消耗
- 方便后续扩展为 SaaS 系统
- 方便新增 Agent，例如选品 Agent、定价 Agent、广告 Agent、自动上架 Agent

# 七、Token Plan 设计

为了降低 AI 调用成本，本项目设计了 Token 使用优化方案。

## 1. 轻量任务使用低成本模型

例如：

- 标题翻译
- 卖点改写
- 商品描述生成
- 类目初筛
- 敏感词初步检查
- SEO 关键词生成

这些任务可以使用轻量模型完成，降低 Token 成本。

## 2. 复杂任务使用高能力模型

例如：

- 图片理解
- OCR 结果分析
- 商品合规判断
- 商品卖点提炼
- 商品质量评分
- 多步骤推理决策

这些任务交给能力更强的模型处理，提高准确率。

## 3. 建立缓存机制

系统会对以下内容建立缓存：

- 相同商品链接
- 相同商品标题
- 相同图片 URL
- 相似商品描述
- 重复翻译结果
- 重复合规检查结果
- 重复类目判断结果

避免同一个商品重复消耗 Token。

## 4. 分阶段调用 AI

不是所有商品都直接进入完整 AI 流程。

系统会先进行基础规则判断：

- 商品数据是否完整
- 是否缺少图片
- 是否缺少价格
- 是否明显不适合上架
- 是否为重复商品
- 是否缺少 SKU
- 是否缺少核心属性

只有通过基础检查后，才进入后续 AI 处理流程。

## 5. 记录 Token 消耗

系统会记录每个任务的 Token 消耗，方便后续进行成本分析。

可统计内容包括：

- 每个商品消耗多少 Token
- 每个 Agent 消耗多少 Token
- 每个用户消耗多少 Token
- 每个店铺消耗多少 Token
- 哪类商品 AI 成本最高
- 哪类任务最消耗 Token
- 哪个国家站点内容生成成本最高
